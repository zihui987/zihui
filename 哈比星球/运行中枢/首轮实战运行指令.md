# 首轮实战运行指令

> 版本：v1.1
> 生效日期：2026-06-18
> 适用范围：`E:\哈比星球`
> 架构说明：本轮起采用两岗位架构（夏仁执行 + 夏审核），夏夏为核心执掌人直接调度

## 一、目的

本文件用于把新岗位体系从"文件存在"推进到"真实可跑"。

本轮目标不是追求复杂协同，而是验证以下四件事：

1. 夏夏能否正确判断任务是否可委派、并直接指派给夏仁
2. 夏仁能否按边界正确交付
3. 夏审能否按依据正确审核
4. 整条链能否形成完整留痕

## 二、首轮测试任务要求

首轮任务必须满足：

- 低风险
- 单轮闭环
- 输入明确
- 输出明确
- 不涉及事实层修改
- 不涉及规则层修改

推荐首轮任务类型：

1. 基于既有素材写一版小短文案
2. 基于既有设定生成一版短脚本
3. 基于既有资料整理一版提示词草案
4. 基于既有内容生成一版小红书文案草稿

本轮不建议上来就跑：

- 长篇小说
- 多环节图像/视频链
- 跨多个外部平台的自动化发布
- 多跳委派任务

## 三、固定执行链

首轮测试统一固定为：**夏夏 → 夏仁 → 夏审**，不允许跳过任一环节。

### 执行步骤

执行步骤以夏夏视角统一起步：

### Step 1：夏夏创建任务

1. 夏夏（核心执掌人）判断这是一个可委派的独立子任务
2. 通过 MCP 调用 `create_task`，参数见下方规范
3. 确认 task 已创建，状态为 pending
4. 通过 `update_task_status(task_id, assigned)` 或 `handoff_task(task_id, from=夏夏, to=夏仁)` 指派
5. 在 task log 中附上任务背景和具体要求

### Step 2：夏仁接收执行

1. 夏仁看到自己的 assigned 任务
2. 读取 task log 中的要求
3. 调用工具完成具体工作
4. 将结果写入 task log
5. `update_task_status(task_id, in_review)`

### Step 3：夏审接收审核

1. 夏审看到 in_review 任务
2. 对照任务要求和验收标准审查
3. 通过：`update_task_status(task_id, completed)`，附审核意见
4. 退回：`update_task_status(task_id, returned)`，附问题清单

### Step 4：夏夏确认归档

1. 夏夏看到 completed 状态
2. 确认结果后通知杜哥或做后续处理
3. `update_task_status(task_id, archived)`

### 退回场景

如果夏审标记为 returned：

1. 夏夏查看退回原因
2. 更新 task log 追加新指令
3. `update_task_status(task_id, assigned)` 重新指派给夏仁
4. 夏仁修改后再次提交 in_review

## 四、测试工具接口

### task_flow_mcp_server 7 工具

| 工具 | 用途 |
|------|------|
| create_task | 夏夏创建任务（初始 pending） |
| update_task_status | 流转状态用（assigned/in_review/completed/returned/archived） |
| handoff_task | 夏夏直接指派给夏仁或夏审 |
| get_task | 查看任务详情 |
| list_tasks | 按状态/角色筛选任务列表 |
| add_task_log | 记录每一步的执行日志 |
| get_task_stats | 查看任务统计数据 |

### 状态流转

```
pending → assigned → in_review → completed → archived
                         ↓
                     returned → assigned（夏夏重派）
```

## 五、验收标准

本轮通过的标志：

- [ ] 完整跑通一条：夏夏 create → 夏仁执行 → 夏审审核 → 夏夏归档
- [ ] 每步都有对应 task log 记录
- [ ] task 流转可追溯（通过 get_task 查看）
- [ ] 至少完成一个任务类型（见第二节）

## 六、回滚预案

如果本轮测试出现以下情况，立即停止并回退：

- 夏仁偏离 task scope 执行了未授权操作
- 夏审通过了明显不合格的产出
- 夏夏在委派后又忘记跟踪结果，导致任务悬空
- 系统级错误（MCP 断联、DB 损坏等）

回滚操作：夏夏调用 `update_task_status(task_id, pending)` 收回所有在途任务，重新评估后再执行。
