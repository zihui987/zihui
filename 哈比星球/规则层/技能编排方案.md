# 技能编排方案

## 一、总原则

技能不是为人格服务，而是为岗位服务。

本项目技能只允许围绕核心执掌人 + 两岗位建设：

- 夏夏（核心执掌人）
- 夏仁（执行岗）
- 夏审（审核岗）

本轮借鉴 AgentHub，只引入四类能力：

- 任务契约
- 结果回执
- 审核闸门
- 工具最小授权

不引入新的治理岗位，不引入复杂多 Agent 编排前台。

## 二、技能分组

### 1. 夏夏技能组

目标：稳定派单、稳定绑定依据、稳定转交、稳定归档

建议技能：

|- `dispatch-task`
|- `bind-authority-files`
|- `validate-inputs`
|- `handoff-to-review`
|- `archive-closeout`
|- `archive-gate-check`（归档前四件套校验 + 任务号一致性校验）

### 2. 夏仁技能组

目标：稳定执行、稳定回执、稳定阻塞上报

建议技能：

- `produce-deliverable`
- `task-result-report`
- `correction-mode`（格式性打回后的补正执行模式）
- `writing-mode`
- `xhs-mode`
- `douyin-mode`
- `automation-mode`
- `self-check`

### 3. 夏审技能组

目标：稳定审核、稳定给结论、稳定回指依据

建议技能：

- `review-deliverable`
- `review-task-report`
- `fact-consistency-check`
- `rule-consistency-check`
- `risk-check`

## 三、技能建设纪律

1. 技能命名按岗位或动作命名
2. 不按人格情绪命名
3. 技能必须附输入要求
4. 技能必须附输出要求
5. 技能必须附失败条件
6. 技能必须能回指到事实文件、规则文件、交付物或回执文件

## 四、Hermes 优势用到极致的方式

### 1. 用 Profile 隔离岗位状态

- 夏夏一个状态集合
- 夏仁一个状态集合
- 夏审一个状态集合

### 2. 用 Skills 固化重复工作法

- 派单模板化
- 回执模板化
- 审核模板化
- 归档模板化

### 3. 用 Sessions / logs 做复盘证据

- 出错回看路径
- 出错回看动作
- 出错回看责任点

### 4. 用工具链支撑执行，不取代岗位

- 搜索用于查证
- 文件工具用于读写
- 终端用于自动化
- MCP 用于能力扩展

## 五、禁止的技能建设方向

- 建复杂多人格技能森林
- 让单一岗位同时拥有生产和审核技能闭环
- 把技能写成宽泛空话
- 把历史经验未经验证直接升格为强规则
- 把 `delegate_task` 包装成长期治理主流程
