# 派单模板

> 使用岗位：夏夏
> 版本：v2.0 — 新增 dispatch plan 结构（2026-06-17，借鉴 AgentHub plan_tasks）

## 标准派单单

```md
# 派单单

## 1. 任务编号
- `TASK-YYYYMMDD-NNN`

## 2. 任务目标
- 

## 3. 执行岗位
- 夏仁

## 4. 执行模式
- writing / xhs / douyin / automation / engineering

## 5. depends_on（前置依赖）
- 本任务依赖哪些前置任务完成，逐条填写：
  - task_id：前置任务编号（如 TASK-20260617-001）
  - required_outputs：需要的前置任务 output_id 列表（如 ["prd", "design"]）
- 无依赖则留空

## 6. required_inputs（上游产物引用）
- 如需引用上游产物，逐条填写：
  - input_from：来源任务编号（如 TASK-20260617-001）
  - input_ref：引用的上游 output_id（如 prd）
  - purpose：引用用途（fact_basis / rework_base / review_target）

## 7. expected_outputs（本任务产出声明）
- 逐条填写：
  - output_id：唯一标识（如 prd / design / code / report）
  - output_type：产物类型（document / markdown / script / report / archive / image / video）
  - output_path：输出文件绝对路径
  - required：是否强依赖（true / false）
  - handoff_to：产出后转交给谁（下一岗位代号，如 夏审 / 夏仁）

## 8. 输入文件绝对路径
- 

## 9. 输出文件绝对路径
- 

## 10. 事实依据文件
- 

## 11. 规则依据文件
- 

## 12. 允许修改清单
- 

## 13. 禁止修改清单
- 

## 14. 交付标准
- 

## 15. acceptance_criteria（验收标准）
- 每条必须可审核、可举证
- 验收项建议在派单时提前逐条写清，降低夏仁执行偏差

## 16. blocked_policy（阻塞策略）
- 本任务如遇阻塞，按以下策略处理：
  - blocked_policy：缺输入 / 路径冲突 / 依据冲突 / 权限不足 / 其他
  - fallback：重新派单 / 升级给人 / 挂起 / 跳过（仅非强依赖任务可跳过）
- 不填则默认：升级给人

## 17. 回执要求
- 结果必须附任务回执
- 回执状态为 blocked 时，必须写清阻塞原因和阻塞类型
- blocked 任务不进入审核，由夏夏按 blocked_policy 处理

## 18. 产物路径约束
- 交付物输出绝对路径：`E:\哈比星球\交付层\...\`
- 任务回执输出绝对路径：`E:\哈比星球\交付层\回执\TASK-YYYYMMDD-NNN_任务回执.md`
- 审核结论输出绝对路径：`E:\哈比星球\交付层\审核\TASK-YYYYMMDD-NNN_审核结论.md`
- 以上路径在派单时必须明确写入，不可留空，不可写相对路径

## 19. 审核重点
- 

## 20. 风险与备注
-
```

## DAG 协作示例

适用于多任务流水线（夏夏 → 夏仁A → 夏仁B → 夏审）：

```
TASK-001 → 夏仁产出PRD → expected_outputs: [prd → handoff_to:夏审]
                                ↓
TASK-002 → 夏仁出图 → depends_on: [TASK-001.required_outputs:prd]
                        expected_outputs: [design → handoff_to:夏审]
                                ↓
TASK-003 → 夏审审查   → depends_on: [TASK-002.required_outputs:design]
```

每个任务独立编号、独立回执、独立审核。夏夏在全部完成后汇总。

## 使用纪律

- 必须写绝对路径
- 不允许写"按之前规则"
- 不允许写"你自己判断"
- 不允许省略事实依据和规则依据
- 当前现行任务编号统一采用 `TASK-YYYYMMDD-NNN`，不得新发 `HABI-P*` 等历史试跑编号
- 依据单一且无歧义时，夏夏可自动绑定事实和规则文件
- 存在多候选文件、权威冲突、事实层或规则层修改风险时，必须退回补件
- 未附回执要求的派单，视为不合格派单
- 验收项（acceptance_criteria）建议在派单时提前逐条写清，降低夏仁执行偏差
- 归档前必须输出归档前核对单，核对通过后才允许生成归档单
- **depends_on 形成循环依赖时，夏夏必须重新规划，不得强行派单**
- **handoff_to 链必须完整：上游产出 → 下游接收，缺环时报夏夏补充**
- **输入文件路径错误即阻塞：派单中的输入文件绝对路径不存在、写错或无法访问时，夏仁必须立即阻塞退回，不得自行搜索替代文件，不得凭文件名相似度推断，待夏夏补正路径后才能继续**
